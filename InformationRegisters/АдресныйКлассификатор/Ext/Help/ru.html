Регистр сведений "Адресный классификатор"

Регистр сведений "Адресный классификатор" предназначен для хранения адресных элементов - областей, районов, городов, населенных пунктов, улиц, домов, кодов  ИМНС, из которых составляются адреса внутри РБ.

Адресный классификатор, таким образом, предназначен для правильного и быстрого заполнения адреса. При заполнении адресов, например, в контактной информации, из адресного классификатора можно выбрать необходимые адресные элементы и на основе их заполнить адресные поля.

Адрес в пределах Республики Беларусь разбивается на территорию столицы (Минск) и территорию областей (6 областей). Последние делятся на территории районов и городов областного подчинения как административно-территориальных единиц. Территория города при необходимости делится на районы в городе. Территория района делится на территории сельсоветов, поселков городского типа и городов районного подчинения. В состав территории сельсовета входят территории поселков городского типа, территории сельских населенных пунктов и другие территории.  

Адресные элементы могут быть введены вручную или загружены из внешнего файла (сайт Министерства по налогам и сборам Республики Беларусь (www.nalog.gov.by).

Вручную адресный элемент можно ввести в форме списка регистра сведений "Адресный классификатор". Информация в адресный элемент вводится через форму записи регистра сведений.

При вводе вручную информации адресного элемента необходимо указать 14 - значный код, наименование, сокращение, тип и код ИМНС.

Код адресного элемента в свою очередь является позиционным и состоит (справа налево):

СС РРР ГГГ ППП УУУУ ДДДД, КККК, где

СС – код региона,

РРР– код района,

ГГГ – код города,

УУУУ – код улицы,

ДДДД – код дома,

КККК – код квартиры.

Наименование адресного элемента содержит часть адреса, которой заполняется соответствующее адресное поле. Наименование дополняется сокращением, которое можно выбрать из вспомогательного справочника "Адресные сокращения" (в каталоге шаблонов, как дополнительный файл поставки).

Тип адресного элемента - это понятное для пользователя расшифровка уровня адресного классификатора. Указывая в качестве типа "Регион", пользователь фактически устанавливает для адресного элемента уровень в единицу.

Кроме вручную введенных элементов, можно воспользоваться уже готовым адресным классификатором.  Загружают адресные классификаторы посредством формы загрузки адресного классификатора. Для вызова её служит кнопка "Загрузить классификатор" в форме списка адресного классификатора или кнопка "Классификатор" в форме записи адреса регистра сведений "Контактная информация".

Информация для загрузки размещается в файлах таблиц, подготовленных специальным образом. Загрузка настраивается: необходимо указать кодировку файлов, формат представления адресной информации в них. Кроме этого, возможен вариант частичной загрузки классификатора - для этого на закладке "Фильтр по регионам" нужно в списке пометить те из регионов, адресную информацию которых требуется загрузить. Адресная информация непомеченных регионов загружаться не будет.

Важно!!! Чем меньше будет загружено областей (регионов), тем быстрее произойдет загрузка готового классификатора и тем быстрее будет работать выбор адресных элементов.

Отдельно предусмотрена возможность загрузки областей (регионов) из внутренних таблиц информационный базы. Это требуется чтобы установить фильтр по областям (регионам).

Для загрузки необходимы:

  • city.dbf - адресный классификатор (содержит информацию о областях, районах, городах и нас. пунктах);
  • nnul.dbf - классификатор улиц по населенным пунктам;  
  • adrsocr.dbf - классификатор адресных сокращений. Если он не загружен или загружен неправильно, механизм адресного классификатора в некоторых случаях будет работать неверно.
